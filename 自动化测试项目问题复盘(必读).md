## 二、详细问题分析与解决方案

### 1. pip 版本过旧警告

**现象**：

text

```
WARNING: You are using pip version 21.1.1; however, version 26.0.1 is available.
```



**原因**：Python 虚拟环境中 pip 版本过低，可能导致安装依赖时出现兼容性问题。

**解决方案**：

bash

```
python -m pip install --upgrade pip
```



**经验教训**：创建虚拟环境后，第一时间升级 pip 是个好习惯。

------

### 2. `New-Item` 命令在 cmd 中无法识别

**现象**：

text

```
'New-Item' 不是内部或外部命令
```



**原因**：`New-Item` 是 PowerShell 专属命令，在 Windows 命令提示符（cmd）中无法使用。

**解决方案**：

- 切换到 PowerShell 环境
- 或使用 cmd 原生命令：`mkdir` 创建目录，`type nul > file.txt` 创建文件

**经验教训**：注意区分命令行环境，复制命令前确认当前 Shell 类型。

------

### 3. Playwright 访问网站报 `net::ERR_CONNECTION_RESET`

**现象**：

text

```
playwright._impl._errors.Error: Page.goto: net::ERR_CONNECTION_RESET at https://www.saucedemo.com/
```



**原因**：

- 本地网络代理或 VPN 干扰
- 防火墙/杀毒软件拦截 Playwright 浏览器进程
- 网站本身临时不可用或 TLS 握手失败

**解决方案**：

- 显式禁用代理：`browser = p.chromium.launch(proxy=None)`
- 添加重试机制，增加超时时间
- 关闭防火墙/杀毒软件测试
- 使用 `ignore_https_errors=True`
- 更换网络环境（如手机热点）

**经验教训**：自动化测试环境应保持干净，避免代理干扰；关键操作应加入重试逻辑提高稳定性。

### 5. `ModuleNotFoundError: No module named 'allure'`

**现象**：运行测试时提示找不到 allure 模块。

**原因**：未安装 `allure-pytest` 插件。

**解决方案**：

bash

```
pip install allure-pytest
```



**经验教训**：Allure 报告需要两个组件：`allure-pytest`（Python 插件）和 Allure 命令行工具（用于生成 HTML）。CI 环境中也必须安装该插件。

------

### 6. `pytest --headed=false` 参数错误

**现象**：

text

```
pytest: error: argument --headed: ignored explicit argument 'false'
```



**原因**：`pytest-playwright` 插件的 `--headed` 是一个开关参数，不接受 `true/false` 值。直接写 `--headed` 表示有头模式，不写表示无头模式。

**解决方案**：

- 无头模式（CI 推荐）：`pytest --browser=chromium`
- 有头模式（本地调试）：`pytest --browser=chromium --headed`

**经验教训**：使用第三方插件前应阅读其参数定义，避免想当然地传递值。

------

### 7. Git push 报 `SSL_read: Connection was reset, errno 10054`

**现象**：

text

```
fatal: unable to access 'https://github.com/...': OpenSSL SSL_read: Connection was reset, errno 10054
```



**原因**：Windows 环境下网络不稳定、代理干扰或防火墙拦截导致 HTTPS 连接被重置。

**解决方案**（按优先级）：

1. 重试 `git push`
2. 切换为 SSH 协议：`git remote set-url origin git@github.com:用户名/仓库.git`
3. 临时关闭代理或为 Git 配置代理
4. 增加 HTTP 缓冲区：`git config --global http.postBuffer 524288000`
5. 临时禁用 SSL 验证（不推荐）：`git config --global http.sslVerify false`

**经验教训**：HTTPS 协议对网络波动敏感，配置 SSH 密钥可从根本上避免此类问题。

------

### 8. CI 中报 `unrecognized arguments: --alluredir`

**现象**：

text

```
pytest: error: unrecognized arguments: --alluredir=./reports/allure-results --clean-alluredir
```



**原因**：`pytest.ini` 中的 `addopts` 配置了 Allure 参数，但 CI 环境未安装 `allure-pytest` 插件，导致 pytest 无法识别这些参数。

**解决方案**：在 CI 工作流中安装 `allure-pytest`：

yaml

```
- run: pip install pytest pytest-playwright allure-pytest
```



**经验教训**：CI 环境必须与本地开发环境保持依赖一致；任何通过 `addopts` 添加的参数，其依赖插件都必须在运行环境中存在。

## 三、改进措施与最佳实践

### 1. 环境标准化

- 使用 `requirements.txt` 或 `pyproject.toml` 锁定所有依赖版本（包括 `allure-pytest`）
- CI 中直接 `pip install -r requirements.txt`

### 2. 测试脚本健壮性

- 对网络请求类操作（如 `page.goto`）封装重试机制
- 避免硬编码单一测试网站，支持配置化切换

### 3. Git 操作规范

- 推荐使用 SSH 协议与 GitHub 交互，减少网络问题
- 提交前执行 `git status` 确认变更范围

### 4. CI 配置检查清单

- 所有必需的 Python 包都已安装（包括报告插件）
- 命令行参数与插件能力匹配（如 `--headed` 不带值）
- 浏览器驱动已安装（`playwright install chromium`）
- 若生成报告，需上传 artifacts 以便下载查看

### 5. 调试技巧

- 本地先用 `headed=True` 观察执行过程
- CI 失败时可启用 `--tracing=retain-on-failure` 和 `--video=retain-on-failure` 获取现场信息
