# 自动化测试框架搭建完整指南

**项目名称**：Playwright + Pytest Web 自动化测试
**适用人群**：测试初学者 / 转行小白
**最后更新**：2026年4月

------

## 一、项目目标

构建一个基于 **Python + Playwright + Pytest** 的 Web 自动化测试框架，采用 **页面对象模型（POM）** 设计模式，实现登录功能的自动化测试，并集成 **Allure 测试报告** 和 **GitHub Actions 持续集成**。

## 二、环境搭建（Windows 示例）

### 2.1 安装 Python

1. 访问 [python.org](https://python.org/)，下载 Python 3.8 或更高版本。

2. 安装时**务必勾选** “Add Python to PATH”。

3. 验证安装：打开终端（PowerShell），输入：

   bash

   ```
   python --version
   ```

   

   若显示 `Python 3.x.x` 则成功。

### 2.2 创建项目文件夹与虚拟环境

bash

```
cd Desktop
mkdir playwright-pytest-demo
cd playwright-pytest-demo
python -m venv venv
```



激活虚拟环境：

- Windows：`venv\Scripts\activate`
- Mac/Linux：`source venv/bin/activate`

激活后终端左侧会出现 `(venv)`。

### 2.3 安装 Python 依赖包

bash

```
pip install pytest pytest-playwright allure-pytest
playwright install
```



验证安装：

bash

```
python -c "from playwright.sync_api import sync_playwright; print('成功')"
```



### 2.4 安装 Allure 命令行工具（系统全局）

Allure 用来将测试结果生成 HTML 报告。**注意**：Allure 命令行工具不是 Python 包，需要单独安装在操作系统上。

#### 方法一：手动安装（推荐，无需 scoop）

1. 下载 Allure：访问 https://github.com/allure-framework/allure2/releases，下载最新 `allure-<version>.zip`。
2. 解压到无中文无空格的路径，例如 `C:\allure`。
3. 配置环境变量：
   - 搜索“环境变量” → 编辑系统环境变量 → 点击“环境变量”。
   - 在“系统变量”中找到 `Path`，编辑 → 新建 → 填入 `C:\allure\bin`。
   - 确定保存。
4. 验证：**重新打开终端**，输入 `allure --version`，显示版本号即成功。

#### 方法二（备选）：使用 Chocolatey 或 WinGet

- Chocolatey：`choco install allure`（管理员 PowerShell）
- WinGet：`winget install allure`（管理员 PowerShell）

### 2.5 安装 Git（用于版本控制）

1. 访问 [git-scm.com](https://git-scm.com/download/win) 下载安装。

2. 安装后配置用户名和邮箱：

   bash

   ```
   git config --global user.name "你的GitHub用户名"
   git config --global user.email "你的GitHub邮箱"
   ```

### 4.2 页面对象模型（POM）

#### `pages/base_page.py` – 基类

python

```
from playwright.sync_api import Page, expect
import allure

class BasePage:
    def __init__(self, page: Page):
        self.page = page

    def navigate_to(self, url: str):
        self.page.goto(url)

    def click(self, selector: str):
        self.page.click(selector)

    def fill_text(self, selector: str, text: str):
        self.page.fill(selector, text)

    def get_text(self, selector: str) -> str:
        return self.page.text_content(selector)

    def take_screenshot(self, name: str):
        screenshot = self.page.screenshot()
        allure.attach(screenshot, name=name, attachment_type=allure.attachment_type.PNG)
```



#### `pages/login_page.py` – 登录页

python

```
from pages.base_page import BasePage
from playwright.sync_api import Page

class LoginPage(BasePage):
    USERNAME_INPUT = '#username'
    PASSWORD_INPUT = '#password'
    LOGIN_BUTTON = 'button[type="submit"]'
    ERROR_MESSAGE = 'div.alert.alert-danger'

    def __init__(self, page: Page):
        super().__init__(page)

    def open(self):
        self.navigate_to("https://practice.expandtesting.com/login")
        return self

    def enter_username(self, username: str):
        self.fill_text(self.USERNAME_INPUT, username)
        return self

    def enter_password(self, password: str):
        self.fill_text(self.PASSWORD_INPUT, password)
        return self

    def click_login(self):
        self.click(self.LOGIN_BUTTON)
        from pages.secure_page import SecurePage
        return SecurePage(self.page)

    def login(self, username: str, password: str):
        self.open()
        self.enter_username(username)
        self.enter_password(password)
        return self.click_login()

    def get_error_message(self) -> str:
        return self.get_text(self.ERROR_MESSAGE)
```



#### `pages/secure_page.py` – 登录成功页

python

```
from pages.base_page import BasePage
from playwright.sync_api import expect

class SecurePage(BasePage):
    SUCCESS_MESSAGE = 'div.alert.alert-success'
    LOGOUT_BUTTON = 'a:has-text("Logout")'

    def verify_login_success(self):
        expect(self.page.locator(self.SUCCESS_MESSAGE)).to_be_visible()
        expect(self.page.locator(self.SUCCESS_MESSAGE)).to_contain_text("You logged into a secure area!")
        return self
```



#### `tests/test_login.py` – 使用 POM 的测试用例

python

```
import pytest
from pages.login_page import LoginPage

class TestLogin:
    def test_valid_login(self, page):
        login_page = LoginPage(page)
        secure_page = login_page.login("practice", "SuperSecretPassword!")
        secure_page.verify_login_success()

    def test_invalid_login(self, page):
        login_page = LoginPage(page)
        login_page.open()
        login_page.enter_username("wrong_user")
        login_page.enter_password("wrong_password")
        login_page.click_login()
        error = login_page.get_error_message()
        assert "Your username is invalid!" in error
```

#### `tests/test_data_driven.py`

python

```
import pytest
import json
import os
from pages.login_page import LoginPage

def load_data():
    path = os.path.join(os.path.dirname(__file__), '..', 'data', 'test_data.json')
    with open(path, 'r', encoding='utf-8') as f:
        return json.load(f)

class TestDataDriven:
    @pytest.mark.parametrize("user", load_data()["valid_users"])
    def test_valid_users(self, page, user):
        login_page = LoginPage(page)
        secure_page = login_page.login(user["username"], user["password"])
        secure_page.verify_login_success()

    @pytest.mark.parametrize("user", load_data()["invalid_users"])
    def test_invalid_users(self, page, user):
        login_page = LoginPage(page)
        login_page.open()
        login_page.enter_username(user["username"])
        login_page.enter_password(user["password"])
        login_page.click_login()
        assert user["expected"] in login_page.get_error_message()
```

```
[pytest]
addopts = 
    --headed
    --browser chromium
```

## 六、推送到 GitHub（完整流程及错误解决）

### 6.1 在 GitHub 网页上创建空仓库

1. 登录 GitHub，点击右上角 `+` → `New repository`。
2. Repository name 输入 `playwright-pytest-demo`。
3. **不要勾选** “Add a README file”、“Add .gitignore” 或 “Choose a license”。
4. 点击 `Create repository`。

### 6.2 本地推送代码（正确步骤）

打开终端（PowerShell 或 Git Bash），确保当前目录是项目根目录，并且已激活虚拟环境（可选，不强制）。

bash

```
# 1. 初始化 Git 仓库
git init

# 2. 添加所有文件到暂存区
git add .

# 3. 提交到本地仓库（必须！否则没有分支）
git commit -m "初始化自动化测试项目"

# 4. 关联远程仓库（替换为自己的用户名）
git remote add origin https://github.com/你的用户名/playwright-pytest-demo.git

# 5. 推送（注意分支名）
git push -u origin main
```



### 6.3 常见错误及解决方法

#### ❌ 错误：`error: src refspec main does not match any`

**原因**：本地没有任何提交（`git commit` 未执行）或者默认分支名不是 `main`（可能是 `master`）。

**解决方法**：

- 先执行 `git commit`（如上述步骤3）。

- 检查分支名：`git branch`。

  - 如果显示 `* master`，则推送命令改为：

    bash

    ```
    git push -u origin master:main
    ```

    

    或者先重命名分支：

    bash

    ```
    git branch -m master main
    git push -u origin main
    ```

    

#### ❌ 错误：`failed to push some refs to ...`

**原因**：远程仓库已有文件（比如创建时勾选了 README）。

**解决方法**：

bash

```
git pull --rebase origin main
git push origin main
```



#### ❌ 错误：`remote: Repository not found.`

**原因**：远程仓库地址错误。

**解决方法**：

- 从 GitHub 仓库页面复制 `.git` 结尾的地址，重新设置：

  bash

  ```
  git remote set-url origin https://github.com/你的用户名/playwright-pytest-demo.git
  ```

  

### 6.4 日常更新代码的三步

以后每次修改代码后，只需：

bash

```
git add .
git commit -m "描述修改内容"
git push
```

## 七、GitHub Actions 持续集成（可选）

在项目根目录创建 `.github/workflows/playwright.yml`：

yaml

```
name: Auto Tests

on:
  push:
    branches: [ main ]
  workflow_dispatch:

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install pytest pytest-playwright
      - run: playwright install chromium
      - run: pytest --browser=chromium --headed=false
```



推送后，每次提交都会自动运行测试，可在 GitHub 仓库的 Actions 标签页查看结果。

## 八、常见问题与注意事项总结

| 问题现象                                       | 可能原因                      | 解决方案                                                     |
| :--------------------------------------------- | :---------------------------- | :----------------------------------------------------------- |
| `pytest` 命令找不到                            | 未激活虚拟环境                | 执行 `venv\Scripts\activate`                                 |
| `allure` 命令找不到                            | Allure 未安装或环境变量未配置 | 手动安装并添加 `bin` 目录到 PATH                             |
| 测试失败：元素找不到                           | 网址或选择器变化              | 检查测试网站是否可用，更新选择器                             |
| 推送报错 `src refspec main does not match any` | 没有 commit 或分支名不对      | 先 `git commit`，检查 `git branch`，用 `master:main` 推送    |
| 推送时要求输入用户名密码                       | 未配置凭据                    | 使用 GitHub 个人访问令牌（Settings → Developer settings → Personal access tokens） |
| GitHub Actions 测试失败                        | CI 中缺少浏览器驱动           | 添加 `playwright install chromium --with-deps`               |

## 九、曾犯的错误记录

1. **忘记执行 `git commit`**：导致本地没有分支，推送时报 `src refspec main does not match any`。
   **教训**：`git commit` 是创建分支提交历史的必要步骤。
2. **本地分支名为 `master` 却尝试推送到 `main`**：Git 默认分支名可能是 `master`（旧版）或 `main`（新版）。
   **教训**：推送前用 `git branch` 确认分支名，不一致时使用 `git push -u origin master:main` 或重命名分支。
3. **Allure 命令行安装困惑**：误以为 `pip install allure-pytest` 就是 Allure 工具本身。
   **教训**：`allure-pytest` 是 Python 插件，负责生成数据；`allure` 命令是独立工具，需单独安装到系统。
4. **手动安装 Allure 后仍提示找不到命令**：没有重新打开终端，或没有配置环境变量。
   **教训**：安装后必须 **新开终端** 才能加载 PATH 变化。
